# To-HTML: 콘텐츠를 단일 HTML 파일로 시각화 — 가치가 있을 때만

> *"HTML 이 새로운 마크다운이다."* — Tariq Sipha (Anthropic Claude Code 팀)
> 단, Pascal Filiteri 측정: 같은 코드 리뷰가 **마크다운 1,140 토큰 / HTML 5,480 토큰 — 약 5배**.

이 스킬은 콘텐츠를 **단일 self-contained HTML 파일** 로 변환하되, *시각화가 실제로 가치를 더하는 경우에만* 진행하고, 그렇지 않으면 마크다운(혹은 다른 가벼운 형식)으로 돌려보낸다. **수동 스킬** — 사용자가 명시적으로 호출할 때만 실행.

산출물은 항상 단일 파일. 빌드 단계 없음, 서버 없음. CDN 의존성만 사용해 더블클릭으로 열리도록.

---

## HTML 이 적절한 경우 (세 가지 패턴)

다음 세 패턴 중 하나에 들어맞을 때 HTML 을 쓴다. 그 외에는 마크다운이 거의 항상 더 싸고 더 명확하다.

| 패턴 | 무엇을 하나 | 사용자가 쓸 법한 한 줄 |
|------|-------------|------------------------|
| **Compare (비교)** | N 개 옵션을 그리드로 한 화면에 펼쳐 한눈에 비교. 디자인, 카피 변형, 레이아웃, 계획안. | *"이거 HTML 그리드로 펼쳐 줘. 나란히 비교할 수 있게."* |
| **Tune (튜닝)** | 슬라이더·입력·드래그 같은 라이브 컨트롤을 넣어 값을 조작하고 결과값을 복사할 수 있게. | *"값 조정 가능한 슬라이더 넣어 줘. 마음에 드는 값은 복사 버튼으로."* |
| **Learn (학습)** | 다이어그램 + 주석 달린 코드 + 주의 사항을 한 스크롤 안에 묶는 학습 페이지. 개념 설명. | *"다이어그램이랑 주석 달린 코드, 주의할 점 한 HTML 로 정리해 줘."* |

**아래 경우는 마크다운으로 보낼 것:**

- 단일 개념의 평문 위주, 드러낼 구조가 없는 콘텐츠.
- 사용자가 *다시 편집할* 체크리스트·할 일 (마크다운은 편집형, HTML 은 읽기형).
- 단순 코드 스니펫 (에디터의 syntax highlight 으로 충분).
- 500 토큰 미만 — HTML 부가 오버헤드가 콘텐츠보다 크다.

## 워크플로우

### Phase 1 — 입력 식별

*무엇을* 변환할지 우선순위대로 확인:

1. **스킬 인자** — 파일 경로, glob, 또는 인용 블록.
2. **직전 assistant 응답** — "이거 HTML 로" 처럼 지시어만 있으면 거의 항상 직전 메시지.
3. **현재 선택 / 클립보드** — agent 가 접근 가능한 경우만.
4. **사용자에게 묻기.** 한 번에 통합 질문: *"무엇을 변환할까요? (파일 경로 / 직전 출력 / 여기 붙여넣기)"*. 두 번 묻지 않는다.

식별된 입력을 한 줄 설명 `INPUT_DESC` 로 기록(보고에 사용).

### Phase 2 — 분류 + 추산

한 블록으로 사용자에게 보고:

1. **패턴 적합도** — Compare / Tune / Learn / None 중 어디에 해당. 셋 중 어느 것도 아니면 정직하게 None 으로.
2. **토큰 추산** — 입력 자체 크기(영문 위주: `chars / 4`, 한글·CJK 위주: `chars / 2.5`). 그리고 Pascal Filiteri 측정 기반 **HTML 출력은 마크다운 대비 ~5배** 한 줄 경고.
3. **시각화 가치** — High / Medium / Low. 휴리스틱:

   | 입력 신호 | 가치 |
   |-----------|------|
   | 3개 이상 항목의 구조화된 비교 | High → Compare |
   | 사용자가 *조절해 보고 싶을* 수치 값 | High → Tune |
   | 다이어그램 + 코드가 함께 필요한 개념 | High → Learn |
   | 평탄한 prose 위주 | Low |
   | 500 토큰 미만 | Low (오버헤드 승) |
   | 이미 코드 리스팅 | Low (에디터 승) |

한 블록 예시:

```
입력: `notes/onboarding.md` (~2,800 토큰)
패턴 적합도: Compare (온보딩 변형 6개)
HTML 출력은 동일 마크다운 출력의 ~5배 토큰 (Pascal Filiteri 측정).
시각화 가치: High — 그리드로 6개를 한눈에 비교 가능.

진행하시겠어요?  (1) 예, 생성  (2) 마크다운 요약으로 대체  (3) ASCII 표  (4) 취소
```

### Phase 3 — Gate (진행 여부 결정)

- **가치 = High 이고 입력이 500 토큰 이상** → 사용자가 별도 지시 없으면 Phase 4 로 바로 진행.
- **가치 = Low 또는 패턴 = None** → 대안을 제시하고 **사용자 결정을 기다린다.** 조용히 마크다운으로 fallback 하지 않는다. 마땅한 대안이 없으면 그렇게 말하고 사용자 판단에 맡긴다.

기본 대안 선택지:

- **마크다운 요약** — 콘텐츠를 ~200–400 토큰의 구조화된 마크다운으로 압축.
- **ASCII 표** — 채팅 안에 들어가는 표 데이터일 때.
- **Mermaid 다이어그램만** — 시각화 욕구가 다이어그램 하나뿐일 때.
- **그대로 진행** — 사용자가 원하면 존중.

같은 질문을 두 번 묻지 않는다.

### Phase 4 — 도구 선호

기본 도구 (모두 한 줄 `<head>` 에 들어가는 CDN, install 없음):

- **Tailwind CSS (CDN)** — 레이아웃 + 타이포. 항상 포함.
- **Mermaid (CDN)** — 다이어그램이 들어갈 때만 추가.
- **Chart.js (CDN)** — 차트가 들어갈 때만 추가.
- **Vanilla JS** — Tune 패턴의 슬라이더·드래그·copy-to-clipboard 버튼.

**한 번만** 통합 질문으로 묻기:

```
기본은 Tailwind CDN (+ 필요 시 Mermaid / Chart.js). 선호하는 도구 있으세요?
```

사용자가 다른 CSS 프레임워크, 오프라인용 inline-only, 특정 차트 라이브러리 등을 명시하면 그것을 우선. 응답 없으면 기본 사용.

### Phase 5 — 생성 + 저장

**단일 self-contained `.html` 파일** 생성:

- 한 파일. CDN script/link 태그 외 외부 에셋 없음.
- 모든 script 는 `<body>` 끝에 — 페이지가 먼저 렌더되도록.
- Tune 패턴의 모든 컨트롤 옆에는 **현재 값을 클립보드로 복사하는 버튼** 필수. 사용자가 그 값을 Claude 에 다시 붙여넣을 수 있도록.
- Learn 패턴 구조: 제목 → 다이어그램(Mermaid) → 주석 달린 코드 블록 → 하단 "주의 사항 / Caveats" 섹션.
- Compare 패턴: CSS grid, 옵션당 카드 하나, 카드 높이 동일, 상단에 평가 기준 노출.

저장 경로:

```
<save-dir>/YYYY-MM-DD-HHmm-{짧은-슬러그}.html
```

`<save-dir>` 결정 순서:

1. 환경변수 `$TO_HTML_DIR` 이 설정돼 있으면 그 값, **그렇지 않으면**
2. `<cwd>/.claude/to-html/`.

디렉토리가 없으면 생성. 파일시스템 비호환 문자 제거: `:`, `/`, `\`, `?`, `*`, `"`, `<`, `>`, `|`.

**파일을 자동으로 열지 않는다.** 경로만 보고.

### Phase 6 — 보고

다음 형식 그대로:

```
✓ 저장: <save-dir>/{filename}.html

패턴:        Compare | Tune | Learn
입력:        <입력 한 줄 설명>
사용 도구:   Tailwind CDN [+ Mermaid] [+ Chart.js] [+ 사용자 지정]
추산 출력 토큰: ~<n> (마크다운 동등본은 ~<n/5>)
```

Phase 3 의 대안이 *함께* 생성됐다면 함께 명시(드물지만 사용자가 둘 다 요청한 경우 허용).

## 하지 말 것

- **파일 자동 열기 금지.** 원격·헤드리스 환경 사용자도 있고, 브라우저 자동 실행을 싫어하는 사용자도 있다. 경로만 보고하고 멈춘다.
- **가치 Low 일 때 조용히 마크다운으로 fallback 금지.** 대안을 제시하고 사용자 결정을 기다린다. 판단권은 사용자에게.
- **짧은 단일 문단을 HTML 로 변환 금지.** 5배 토큰 비용이 실제로 발생. 사용자에게 경고하고 마크다운 권장.
- **임의의 npm 패키지·로컬 파일 끌어오기 금지.** 단일 파일 + CDN-only.
- **같은 질문 두 번 금지.** 패턴+추산+대안은 *한 블록*, 도구 선호는 *한 줄*.
- **자동 트리거 금지.** 사용자가 명시적으로 호출할 때만 실행.

## 커스터마이즈

기본값은 의도적으로 단순. 이 SKILL.md 를 직접 편집해 변경:

- **저장 디렉토리** — 환경변수 `TO_HTML_DIR` 사용, 혹은 Phase 5 의 default 수정.
- **기본 도구 셋** — Phase 4 의 CDN 목록. 오프라인이 잦으면 Plain HTML/CSS 를 기본으로 변경하고 여기 명시.
- **슬러그 길이** — 현재 한·영 자연어 기준 40자 이내. 파일명 충돌이 잦으면 Phase 5 에서 조정.
